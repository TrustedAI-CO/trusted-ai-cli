Conventions — tai CLI

Code Style

No formatter config file (.ruff.toml, pyproject [tool.ruff]) is present; the project relies on PEP 8 conventions enforced by review rather than a pre-commit hook. Type annotations are used on all public function signatures. Imports follow a consistent order: stdlib → third-party → internal. All modules begin with a short docstring explaining the module's purpose and any invariants worth knowing (see the http.py module-level comment: "Every command calls build_client(ctx) — never constructs requests directly.").

Logging uses logging.getLogger(__name__) (abbreviated as _log or log) rather than print(). Rich Console objects handle all user-facing output. Each command module creates its own console (stdout) and err_console (stderr) instances rather than sharing global ones.

Error Handling

All domain errors inherit from TaiError(message, hint=None). The hint field carries an actionable fix shown to the user (e.g. "Run: tai auth login"). Subclasses map semantic HTTP status codes to typed exit codes via ExitCode constants (0 = SUCCESS, 1 = ERROR, 2 = USAGE, 3 = NOT_FOUND, 4 = PERMISSION_DENIED, 5 = CONFLICT).

Commands catch TaiError inside their handlers and delegate to handle_error(exc), which prints to stderr and calls raise typer.Exit(exc.exit_code). The top-level cli() entry point catches any uncaught TaiError as a safety net. Never let a TaiError propagate to typer's default exception handler — that path is disabled (pretty_exceptions_enable=False).

Common Patterns

1. AppContext via typer.Context.obj

Runtime state (profile, verbose flag, loaded config) lives in AppContext, a plain dataclass stored in typer.Context.obj. Every command function that needs it calls app_ctx = get_ctx(ctx). Tests inject a pre-built AppContext directly as obj=ctx_obj in runner.invoke(), bypassing config file loading entirely.

2. build_client — single HTTP entry point

Commands never construct httpx.Client directly. They call build_client(app_ctx), which returns a pre-configured client with the bearer token, base URL, JSON headers, and a response event hook that raises ApiError on 4xx/5xx. Token refresh on 401 is handled transparently inside _BearerAuth.auth_flow().

3. Pydantic models for config, frozen dataclasses for value objects

ProfileConfig and TaiConfig are Pydantic BaseModel subclasses, validated with model_validate(). Simpler, immutable value objects (e.g. BrandColors) use @dataclass(frozen=True).

4. JSON output branch in every command

Every command checks app_ctx.json_output and calls console.print_json() with the raw data dictionary when true. The human-readable path uses Rich tables, Markdown, or formatted text. Keep these two branches in sync when adding new fields.

5. Plugin discovery via entry points

External commands register under the tai.plugins entry-point group in their own pyproject.toml. On startup, _load_plugins() discovers and attaches them as Typer sub-apps. Load failures emit a yellow warning and are swallowed — they must never crash the CLI.

Gotchas

  • Tests inject AppContext directly. If you forget obj=ctx_obj in runner.invoke(), the main callback will try to read ~/.config/tai/config.toml on your real filesystem, causing flaky or environment-dependent test results.
  • Coverage is enforced at 80%. pytest fails if coverage drops below that threshold. Sales and browser-setup modules are excluded from coverage because they require optional dependencies (playwright).
  • Config merges project over user, not the other way. A .tai.toml in the current working directory silently overrides ~/.config/tai/config.toml. Tests use monkeypatch to redirect both paths into a tmp_path to avoid this pollution.
  • from __future__ import annotations is inconsistently present. Some modules include it (enabling PEP 563 lazy annotation evaluation), others don't. Avoid relying on runtime annotation introspection (e.g. get_type_hints()) without importing this — or use string literals for forward references.
  • Rich output goes to stdout; errors go to stderr. The CliRunner captures both in result.output by default. Use mix_stderr=False if you need to assert on them independently.
