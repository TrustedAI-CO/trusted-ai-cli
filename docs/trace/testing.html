Testing Trace

Framework and Runner

Tests use pytest 8+ with pytest-cov for coverage, respx and pytest-httpx for mocking httpx calls, and Typer's built-in CliRunner for end-to-end command invocations.

Running Tests

# Full suite with coverage (default — enforces 80% threshold)
uv run pytest

# Single file
uv run pytest tests/test_http.py

# Specific test
uv run pytest tests/commands/test_ai_cmd.py::test_ai_complete

# Skip coverage enforcement
uv run pytest --no-cov

Coverage Configuration

Configured in pyproject.toml under [tool.pytest.ini_options] and [tool.coverage.run]:

  • Source measured: tai/
  • Minimum threshold: 80% — build fails below this
  • Report: term-missing (line numbers of uncovered lines printed inline)
  • Excluded from coverage: tai/core/sales/, tai/commands/sales.py, tai/core/sales/aimitsu.py, tai/core/sales/hnavi.py (require optional playwright dependency)

Test Layout

Tests mirror the source tree under tests/:

  • tests/ — top-level unit tests for core modules (test_http.py, test_config.py, test_errors.py, etc.)
  • tests/commands/ — CLI integration tests, one file per command module
  • tests/core/ — additional unit tests for core utilities (test_mermaid.py, test_style.py, test_brand_colors.py)
  • tests/conftest.py — shared fixtures: runner, default_profile, app_ctx

Testing Philosophy

Command tests invoke the real Typer app via CliRunner.invoke(app, [...], obj=ctx_obj), injecting a pre-built AppContext to bypass filesystem config loading. HTTP calls are mocked by patching build_client to return a MagicMock, or by using @respx.mock decorators for lower-level httpx transport tests.

The fixture pattern repeats across command test files: construct a minimal TaiConfig with a ProfileConfig, wrap it in an AppContext, pass it as obj. Conftest provides a default version; override locally when you need specific field values.

Config tests use monkeypatch to redirect CONFIG_DIR, CONFIG_FILE, and PROJECT_CONFIG_FILE into tmp_path, ensuring no test touches real user config on disk.
