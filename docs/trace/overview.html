tai CLI — Codebase Overview

What it does

tai is the TrustedAI internal CLI — a Python tool built with Typer that centralises the company's developer toolchain. It authenticates engineers with Google OAuth (enforcing the company domain), manages secrets in the OS keychain, installs AI-agent skills and quality-gate hooks into Claude Code and Codex, compiles branded PDFs from Markdown, manages workspace projects and tasks through a Hub API, and provisions on-demand GPU instances on vast.ai for ML workloads. Every command surfaces a --json flag for scripting, and all commands share one AppContext carrying the active profile, verbosity, and loaded config.

Key concepts

  • CLI Entrypoint & AppContext — Typer wires every subcommand together; a single AppContext object is built once in the global callback and threaded through every command via typer.Context.obj.
  • Authentication — Two independent OAuth flows: Google PKCE (company login) and Claude.ai PKCE (Claude Code login); tokens live in the keychain, never on disk unprotected.
  • Configuration & Profiles — TOML-based config with XDG paths; user-level and project-level (.tai.toml) files are deep-merged; named profiles allow switching between dev / staging / prod.
  • Secret Management — Three-layer fallback: OS keychain → encrypted JSON file (CI) → TAI_SECRET_* env vars; never written to flags or plain config.
  • Skills & Hooks — Bundled Claude Code / Codex skills (Markdown files with YAML front matter) are discovered and copied to ~/.claude/skills/; hooks are Node.js scripts wired into ~/.claude/settings.json to run quality gates and session tracking automatically.
  • PDF Generation — Markdown or Typst source is compiled to branded PDF via the typst CLI; Typst templates ship as package data and are installed to ~/.tai/templates/; brand colours come from a brand.toml.
  • Hub API — REST client for the internal Hub service; commands manage workspace projects, pages, tasks, and members; auth uses the Google id_token from the keychain as a Bearer token.
  • GPU Provisioning (vast.ai) — Wraps the vastai CLI to find the cheapest matching GPU offer, create an SSH keypair, boot an instance, sync the user's repo, and tear it back down; state is persisted locally so aliases survive restarts.
